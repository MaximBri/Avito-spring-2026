# О проекте

Веб-приложение для создания и редактирования объявлений с AI-ассистентом, который помогает улучшить описание и цену.

Основная цель — упростить процесс публикации объявлений и повысить их качество.

## Основные возможности

---

### Работа с объявлениями
Пользователь может:
- просматривать список объявлений
- искать по названию
- применять фильтры и сортировки (включая сортировку по цене)
- переходить к детальному просмотру объявления

---

### Просмотр объявления
- отображение полной информации
- удобная навигация назад
- переход к редактированию

---

### Редактирование объявления

Форма динамически подстраивается под категорию и включает:
- обязательные и необязательные поля
- визуальную подсветку:
  - красные обязательные поля (с ошибками)
  - оранжевые необязательные, требующие внимания

---

### AI-ассистент (llama3)

Интегрирован AI, который помогает пользователю:
- улучшить описание объявления
- скорректировать цену

Особенности:
- работает прямо в форме
- пользователь контролирует результат (может принять или отклонить изменения)
- повышает качество объявлений без дополнительных усилий

---

### Работа с черновиками на странице редактирования объявления

Реализовано автосохранение формы:
- данные автоматически сохраняются в `localStorage`
- при повторном открытии формы:
  - показывается модальное окно
  - пользователь может продолжить редактирование черновика
  - или начать с чистого состояния

## Структура страниц

- `/` → редирект на `/ads`
- `/ads` → список объявлений
- `/ads/:id` → просмотр объявления
- `/ads/:id/edit` → редактирование объявления

## Технологический стек фронтенда:

- React
- TypeScript
- Vite
- Axios
- Tanstack query
- React-hook-form
- Zod
- Mantite

## Решения, принятые самостоятельно:

1) Перенастроена валидация характеристик на бэкенде
2) Настроен CORS для корректного взаимодействия клиента и сервера
3) Реализована сортировка по цене на стороне бэкенда
4) Переделана бизнес-логика флага `needsRevision`:
  - `true` — если есть незаполненные необязательные поля
  - `false` — если форма заполнена полностью

## Запуск проекта

Проект можно запустить двумя способами:

* через Docker (рекомендуется)
* вручную (frontend + backend отдельно)

---

## Запуск через Docker (рекомендуется)

### 1. Требования

* Установленный Docker

---

### 2. Запуск

В корне проекта выполнить:

```bash
docker compose up -d --build
```

---

### 3. Доступные сервисы

После запуска:

* Frontend: http://localhost:5173/
* Backend: http://localhost:8080/
* AI (Ollama / llama3): http://localhost:11434/

---

## Локальный запуск (без Docker)

### 1. Подготовка

В директориях `client` и `server` создать `.env` файлы на основе `.env-example`

---

## Backend

Перейти в папку `server`:

```bash
cd server
npm install
npm run start
```

Backend будет доступен на:
http://localhost:8080/

---

## Frontend

Перейти в папку `client`:

```bash
cd client
npm install
npm run dev
```

Frontend будет доступен на:
http://localhost:5173/

---

## AI (Ollama)

Если запускаешь без Docker, нужно отдельно:

1. Установить Ollama
2. Запустить:

```bash
ollama serve
ollama pull llama3
```

---

## Важно

Frontend использует переменные окружения:

* `VITE_BACKEND_URL` — URL бэкенда
* `VITE_OLAMA_URL` — URL Ollama

Убедись, что они корректно указаны в `.env`
